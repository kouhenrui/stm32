# 硬件连接说明

本文档描述本工程所有外设模块与 **STM32F103C8** 开发板的接线方式。引脚定义以 [`include/pin_config.h`](../include/pin_config.h) 为准；若改线请同步修改该文件与 [`openspec/specs/hardware/spec.md`](../openspec/specs/hardware/spec.md)。

## 模块一览

| 模块 | 接口 | STM32 引脚 | 说明 |
|------|------|------------|------|
| 0.9" OLED（SSD1306） | I2C1 | PB6（SCL）、PB7（SDA） | 3.3V，地址 0x3C |
| ESP8266 WiFi | USART2 | PA2（TX）、PA3（RX） | 3.3V，115200，交叉连接 |
| 蜂鸣器（有源） | GPIO | PB0 | 高电平响 |
| 调试串口（可选） | USART1 | PA9（TX）、PA10（RX） | 115200 |
| ST-Link 烧录 | SWD | SWDIO、SWCLK | 调试与烧录 |

## 总体接线示意

```
                    ┌─────────────────────┐
                    │   STM32F103C8       │
                    │                     │
    OLED SCL ───────┤ PB6 (I2C1_SCL)      │
    OLED SDA ───────┤ PB7 (I2C1_SDA)      │
    蜂鸣器控制 ──────┤ PB0                 │
                    │                     │
    ESP8266 RX ─────┤ PA2 (USART2_TX)     │  MCU TX → 模块 RX
    ESP8266 TX ─────┤ PA3 (USART2_RX)     │  MCU RX ← 模块 TX
                    │                     │
    USB-TTL RX ─────┤ PA9  (USART1_TX)    │  可选调试
    USB-TTL TX ─────┤ PA10 (USART1_RX)    │
                    │                     │
    ST-Link SWDIO ──┤ SWDIO               │
    ST-Link SWCLK ──┤ SWCLK               │
                    │                     │
    3.3V ───────────┤ 3.3V ────┬── OLED VCC
                    │          ├── ESP8266 VCC / EN / GPIO0
                    │          └── 蜂鸣器模块 VCC（视模块而定）
    GND ────────────┤ GND ─────┴── 所有模块 GND 共地
                    └─────────────────────┘
```

---

## 1. 电源与共地

| 要求 | 说明 |
|------|------|
| 工作电压 | STM32、OLED、ESP8266 均为 **3.3V** 逻辑电平 |
| 共地 | 所有模块 **GND 必须相连** |
| ESP8266 电流 | 峰值可达 300mA+，建议独立 3.3V 稳压或加大 LDO 容量，勿仅靠 USB 弱供电 |
| OLED | **禁止接 5V**，仅 3.3V |

---

## 2. OLED 显示屏（0.9" SSD1306，I2C）

适用于四线 I2C 模块（VCC、GND、SCL、SDA）。

| OLED 引脚 | 接 STM32 | 备注 |
|-----------|----------|------|
| VCC | 3.3V | |
| GND | GND | |
| SCL | **PB6** | I2C1 时钟 |
| SDA | **PB7** | I2C1 数据 |

### 补充说明

- I2C 设备地址：**0x3C**（部分模块为 0x3D，不亮时修改 `OLED_I2C_ADDRESS`）
- 模块通常自带 SCL/SDA 上拉电阻；若无，各接 **4.7kΩ** 到 3.3V
- 本工程 **不支持 SPI 接口** OLED；若为 SPI 模块需更换驱动

---

## 3. 蜂鸣器（有源）

固件假定 **有源蜂鸣器**，`PB0` 输出高电平时发声（见 `BUZZER_ACTIVE_LEVEL`）。

### 3.1 有源蜂鸣器模块（推荐）

| 模块引脚 | 接 STM32 | 备注 |
|----------|----------|------|
| VCC | 3.3V 或 5V | 按模块丝印 |
| GND | GND | |
| I/O、S、SIG | **PB0** | 控制端 |

### 3.2 裸有源蜂鸣器（加三极管驱动）

```
PB0 ── 1kΩ ── NPN 基极（如 S8050）
              集电极 ── 蜂鸣器负极
              发射极 ── GND
蜂鸣器正极 ── 3.3V / 5V（按蜂鸣器规格）
```

建议在蜂鸣器两端反向并联二极管（如 1N4148），吸收关断反电动势。

### 3.3 无源蜂鸣器

当前固件 **不适用** 无源蜂鸣器（需 PWM 方波）。若使用无源蜂鸣器，须修改 `lib/buzzer/` 驱动。

---

## 4. ESP8266 WiFi 模块

通过 **USART2** 以 AT 指令通信，波特率 **115200**。TX/RX 须 **交叉连接**。

| ESP8266 引脚 | 接 STM32 | 备注 |
|--------------|----------|------|
| VCC | 3.3V | 注意供电能力 |
| GND | GND | |
| TX | **PA3** | 模块发 → MCU 收（USART2_RX） |
| RX | **PA2** | 模块收 ← MCU 发（USART2_TX） |
| EN / CH_PD | 3.3V | 使能，保持高电平 |
| GPIO0 | 3.3V | 正常运行模式（非下载模式） |
| RST | 悬空或 3.3V | 可选，低电平复位 |

### 接线记忆

```
STM32 PA2 (TX) ──────► ESP8266 RX
STM32 PA3 (RX) ◄────── ESP8266 TX
```

### WiFi 配置

SSID 与密码在 [`include/app_config.h`](../include/app_config.h) 中设置。

## 5. 调试串口（USART1，可选）

用于 USB-TTL 查看日志，非必须。

| USB-TTL | 接 STM32 | 备注 |
|---------|----------|------|
| RX | **PA9** | 接 MCU TX |
| TX | **PA10** | 接 MCU RX |
| GND | GND | 必须共地 |

串口参数：**115200**，8N1。

---

## 6. ST-Link 烧录与调试（SWD）

| ST-Link | 接 STM32 | 备注 |
|---------|----------|------|
| SWDIO | SWDIO | 数据线 |
| SWCLK | SWCLK | 时钟线 |
| GND | GND | |
| 3.3V | 3.3V | 可选目标供电 |

烧录命令：

```bash
python -m platformio run --target upload
```

## 7. 引脚总表

| STM32 引脚 | 功能 | 连接目标 |
|------------|------|----------|
| PB6 | I2C1_SCL | OLED SCL |

## 8. 上电自检

按 [`src/main.c`](../src/main.c) 启动流程，正常时应观察到：

| 步骤 | 现象 |
|------|------|
| 上电 | OLED 显示 `STM32F103` / `Booting...` |
| 蜂鸣器 | 短鸣一声（约 150ms） |
| ESP8266 | OLED 显示 `ESP8266` / `AT OK` 或错误码 |
| WiFi | 若已配置 SSID，显示连接结果 |
| 结束 | 再短鸣一声 |

## 9. 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| OLED 不亮 | 接 5V、I2C 接反、地址错误 | 改 3.3V；核对 SCL/SDA；试 0x3D |
| 蜂鸣器不响 | 无源蜂鸣器、PB0 被占用 | 换有源模块；查开发板原理图 |
| ESP8266 无响应 | TX/RX 未交叉、供电不足 | 核对 PA2↔RX、PA3↔TX；加强 3.3V 供电 |
| GPIO0 未接高 | 模块进入下载模式 | GPIO0 接 3.3V |
| 烧录失败 | SWD 线松动 | 检查 SWDIO、SWCLK、GND |
