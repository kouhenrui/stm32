# 硬件连接说明

本文档描述本工程所有外设模块与 **STM32F103C8** 开发板的接线方式。引脚定义以 [`include/pin_config.h`](../include/pin_config.h) 为准；若改线请同步修改该文件与 [`openspec/specs/hardware/spec.md`](../openspec/specs/hardware/spec.md)。

文档编写规范见 [`docs/文档准则.md`](文档准则.md)（总体图 + 分模块图、丝印命名、禁止 § 符号等）。

## 引脚命名说明

开发板排针丝印通常写作 **`A2`、`B6`** 等形式（**无 `P` 前缀**）。  
固件代码中使用 HAL 命名 **`PA2`、`PB6`**（`P` + 端口 + 引脚号），**一一对应**：

| 开发板丝印 | 代码 / 数据手册 | 含义 |
|------------|-----------------|------|
| A2 | PA2 | 端口 A，第 2 脚 |
| B6 | PB6 | 端口 B，第 6 脚 |

下文表格与接线图 **以开发板丝印为准**；括号内标注代码名供对照 `pin_config.h`。

## 模块一览

| 模块 | 接口 | 开发板引脚 | 说明 |
|------|------|------------|------|
| 0.9" OLED（SSD1306） | I2C1 | **B6**（SCL）、**B7**（SDA） | 3.3V，地址 0x3C |
| ESP8266 WiFi | USART2 | **A2**（TX）、**A3**（RX） | 3.3V，115200，交叉连接 |
| 蜂鸣器（有源） | GPIO | **B0** | 高电平响 |
| 调试串口（可选） | USART1 | **A9**（TX）、**A10**（RX） | 115200 |
| ST-Link 烧录 | SWD | SWDIO、SWCLK | 调试与烧录 |

## 总体接线示意图

以下为 **全部外设与开发板** 的汇总关系；各模块细节见后续分节。

```
                         ┌──────────────────────────┐
                         │      STM32F103C8         │
                         │                          │
   OLED SCL ─────────────┤ B6   (代码 PB6, I2C1_SCL) │
   OLED SDA ─────────────┤ B7   (代码 PB7, I2C1_SDA) │
   蜂鸣器 I/O ────────────┤ B0   (代码 PB0)          │
                         │                          │
   ESP8266 RX ◄───────────┤ A2   (代码 PA2, USART2_TX) │  MCU 发 → 模块收
   ESP8266 TX ───────────► A3   (代码 PA3, USART2_RX) │  MCU 收 ← 模块发
                         │                          │
   USB-TTL RX ◄───────────┤ A9   (代码 PA9, USART1_TX) │  可选，调试日志
   USB-TTL TX ───────────► A10  (代码 PA10, USART1_RX)│
                         │                          │
   ST-Link SWDIO ────────┤ SWDIO                    │
   ST-Link SWCLK ────────┤ SWCLK                    │
                         │                          │
   3.3V ─────────────────┤ 3.3V ─────┬── OLED VCC
                         │           ├── ESP8266 VCC / EN / GPIO0
                         │           └── 蜂鸣器 VCC（视模块）
   GND ──────────────────┤ GND ──────┴── 所有外设 GND 共地
                         └──────────────────────────┘
```

> 分模块接线图：**第 1 节** 电源与共地 · **第 2 节** OLED · **第 3 节** 蜂鸣器 · **第 4 节** ESP8266 · **第 5 节** 调试串口 · **第 6 节** ST-Link

## 1. 电源与共地

所有模块必须 **共地**，逻辑电平均为 **3.3V**（OLED 禁止接 5V）。

```
开发板 3.3V ──┬── OLED VCC
              ├── ESP8266 VCC / EN / GPIO0
              └── 蜂鸣器模块 VCC（视模块丝印，部分模块为 5V）

开发板 GND  ──┴── OLED GND
              ├── ESP8266 GND
              ├── 蜂鸣器 GND
              ├── USB-TTL GND（若接调试串口）
              └── ST-Link GND
```

| 要求 | 说明 |
|------|------|
| 工作电压 | STM32、OLED、ESP8266 均为 **3.3V** 逻辑 |
| 共地 | 上图中所有 **GND 必须相连** |
| ESP8266 电流 | 峰值 300mA+，建议独立 3.3V 稳压或加大 LDO，勿仅靠 USB 弱供电 |

## 2. OLED 显示屏（0.9" SSD1306，I2C）

适用于四线 I2C 模块（VCC、GND、SCL、SDA）。

```
OLED VCC ────── 开发板 3.3V
OLED GND ────── 开发板 GND
OLED SCL ────── 开发板 B6    （代码 PB6 / I2C1_SCL）
OLED SDA ────── 开发板 B7    （代码 PB7 / I2C1_SDA）
```

| OLED 引脚 | 接开发板 | 备注 |
|-----------|----------|------|
| VCC | 3.3V | |
| GND | GND | |
| SCL | **B6** | I2C1 时钟 |
| SDA | **B7** | I2C1 数据 |

### 补充说明

- I2C 设备地址：**0x3C**（部分模块为 0x3D，不亮时修改 `OLED_I2C_ADDRESS`）
- 模块通常自带 SCL/SDA 上拉电阻；若无，各接 **4.7kΩ** 到 3.3V
- 本工程 **不支持 SPI 接口** OLED

## 3. 蜂鸣器（有源）

固件假定 **有源蜂鸣器**，**B0** 输出高电平时发声（代码 `PB0`，见 `BUZZER_ACTIVE_LEVEL`）。

### 3.1 有源蜂鸣器模块（推荐）

```
模块 VCC ────── 3.3V 或 5V（按模块丝印）
模块 GND ────── GND
模块 I/O ────── 开发板 B0    （代码 PB0）
```

| 模块引脚 | 接开发板 | 备注 |
|----------|----------|------|
| VCC | 3.3V 或 5V | 按模块丝印 |
| GND | GND | |
| I/O、S、SIG | **B0** | 控制端 |

### 3.2 裸有源蜂鸣器（加三极管驱动）

```
B0 ── 1kΩ ── NPN 基极（如 S8050）
            集电极 ── 蜂鸣器负极
            发射极 ── GND
蜂鸣器正极 ── 3.3V / 5V（按蜂鸣器规格）
```

建议在蜂鸣器两端反向并联二极管（如 1N4148），吸收关断反电动势。

### 3.3 无源蜂鸣器

当前固件 **不适用** 无源蜂鸣器（需 PWM 方波）。若使用无源蜂鸣器，须修改 `lib/buzzer/` 驱动。

## 4. ESP8266 WiFi 模块

通过 **USART2** 以 AT 指令通信，波特率 **115200**。**TX/RX 须交叉连接**。

```
ESP8266 VCC  ────── 开发板 3.3V
ESP8266 GND  ────── 开发板 GND
ESP8266 EN   ────── 3.3V          （使能，保持高电平）
ESP8266 GPIO0 ───── 3.3V          （正常运行，非下载模式）

开发板 A2 ────────► ESP8266 RX    （MCU 发 → 模块收，代码 PA2 / USART2_TX）
开发板 A3 ◄──────── ESP8266 TX    （MCU 收 ← 模块发，代码 PA3 / USART2_RX）
```

| ESP8266 引脚 | 接开发板 | 备注 |
|--------------|----------|------|
| VCC | 3.3V | 注意供电能力 |
| GND | GND | |
| TX | **A3** | 模块发 → MCU 收 |
| RX | **A2** | 模块收 ← MCU 发 |
| EN / CH_PD | 3.3V | 使能 |
| GPIO0 | 3.3V | 正常运行模式 |
| RST | 悬空或 3.3V | 可选，低电平复位 |

### WiFi 配置

SSID 与密码在 [`.env`](../.env.example) 中设置（`WIFI_SSID_1`、`WIFI_PASSWORD_1` 等），编译时注入固件。

## 5. 调试串口（USART1，可选）

用于 USB-TTL 查看日志，非必须。

```
USB-TTL RX ◄──── 开发板 A9     （MCU 发，代码 PA9 / USART1_TX）
USB-TTL TX ────► 开发板 A10    （MCU 收，代码 PA10 / USART1_RX）
USB-TTL GND ───── 开发板 GND
```

| USB-TTL | 接开发板 | 备注 |
|---------|----------|------|
| RX | **A9** | 接 MCU TX |
| TX | **A10** | 接 MCU RX |
| GND | GND | 必须共地 |

串口参数：**115200**，8N1。

## 6. ST-Link 烧录与调试（SWD）

```
ST-Link SWDIO ── 开发板 SWDIO
ST-Link SWCLK ── 开发板 SWCLK
ST-Link GND   ── 开发板 GND
ST-Link 3.3V  ── 开发板 3.3V    （可选，为目标板供电）
```

| ST-Link | 接开发板 | 备注 |
|---------|----------|------|
| SWDIO | SWDIO | 数据线 |
| SWCLK | SWCLK | 时钟线 |
| GND | GND | |
| 3.3V | 3.3V | 可选目标供电 |

烧录命令：

```bash
python -m platformio run --target upload
```

## 8. 上电自检

按 [`src/main.c`](../src/main.c) 启动流程，正常时应观察到：

| 步骤 | 现象 |
|------|------|
| 上电 | OLED 显示 `STM32F103` / `Booting...` |
| ESP8266 | OLED 显示 `ESP8266` / `AT OK` 或错误码 |
| WiFi | 显示 `WiFi OK` 及 SSID，或 `WIFI FAIL` |
| 天气 | 显示上海天气图标、温度、湿度；或 `NO API KEY` / `NET FAIL` |
| 蜂鸣器 | 若 `FEATURE_BUZZER=1`，短鸣提示 |

## 9. 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| OLED 不亮 | 接 5V、I2C 接反、地址错误 | 改 3.3V；核对 B6/B7；试 0x3D |
| 蜂鸣器不响 | 无源蜂鸣器、B0 被占用 | 换有源模块；查开发板原理图 |
| ESP8266 无响应 / AT TIMEOUT | TX/RX 未交叉、EN/GPIO0 未接 3.3V、供电不足、模块未就绪 | A2→模块 RX，A3←模块 TX；EN、GPIO0 接 3.3V；共地；独立 3.3V 供电；重新烧录后等 2s |
| GPIO0 未接高 | 模块进入下载模式 | GPIO0 接 3.3V |
| 烧录失败 | SWD 线松动 | 检查 SWDIO、SWCLK、GND |
| 天气拉取失败 | API Key / WiFi 未配 | 检查 `.env` 中 `WEATHER_API_KEY`、`WIFI_SSID_1` |
| WiFi 账号正确仍失败 | 电脑热点仅 5GHz、WPA3、模块扫不到 SSID | 热点改为 **2.4GHz + WPA2**；OLED 若显示 `NOT IN SCAN` 即模块看不见该热点 |
