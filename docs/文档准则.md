# 项目文档准则

本文档为 **STM32F103 固件工程** 的文档编写与维护规范，适用于 `docs/`、`README.md`、OpenSpec 规格及 AI 辅助生成的说明文字。

## 硬件连接文档

[`硬件连接.md`](硬件连接.md) 须同时包含：

1. **总体接线示意图** — 一张汇总图，展示开发板与全部外设的信号与电源关系
2. **分模块接线示意图** — 每个外设（OLED、蜂鸣器、ESP8266、调试串口、ST-Link 等）单独一节，附独立接线图与表格

改线时须 **同步更新** 总体图与各分模块图，并保持两者一致。

## 引脚命名

| 场景 | 命名方式 | 示例 |
|------|----------|------|
| 面向用户的文档、接线图 | **开发板丝印**（无 `P` 前缀） | `A2`、`B6`、`B0` |
| 固件代码、`pin_config.h`、HAL | **代码名** | `PA2`、`PB6`、`PB0` |

用户文档以丝印为准；必要时在括号内标注代码名供对照，例如：`B6（代码 PB6）`。

## 禁止使用的符号

- **不得使用「§」符号** 表示章节或序号（如「§1」「§2」）。请使用 Markdown 标题（`## 1.`）或中文「第 1 节」等写法。

## 敏感信息

- API Key、WiFi 密码等 **不得写入** 可提交的源码或文档
- 统一放在 `.env`，通过 `extra_scripts/env_config.py` 编译时注入

## 变更同步清单

修改硬件或文档时，按顺序核对：

1. [`include/pin_config.h`](../include/pin_config.h)
2. [`docs/硬件连接.md`](硬件连接.md)（总体图 + 分模块图）
3. [`openspec/specs/hardware/spec.md`](../openspec/specs/hardware/spec.md)
4. [`src/stm32f1xx_hal_msp.c`](../src/stm32f1xx_hal_msp.c)（若更换外设实例）

## OpenSpec 与 README

- 提案与设计文档遵循 [`openspec/config.yaml`](../openspec/config.yaml) 中的 `rules`
- `README.md` 面向快速上手，详细接线以 `docs/硬件连接.md` 为准
